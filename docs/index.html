The Problem

Ruby package management still feels like startup tax.

Rails teams pay for the same friction over and over: cold CLI startup, Bundler overhead, container path mismatch, and dependency tooling that feels split across three eras.

Slow path to first feedback

Parsing, bootstrapping, and basic gem tasks start with Ruby process cost before useful work begins.

Rails workflow fragmentation

`gem`, Bundler, project tasks, and Rails commands all feel related, but they do not live in one coherent CLI.

CI and Docker churn

Environment-specific paths and repeated install work keep slowing down container builds and deploy flows.

Poor observability

When dependencies drift or lockfiles get heavy, the default toolchain rarely explains the problem quickly.

The Solution

One CLI for gem work, Bundler-adjacent flows, and Rails operators.

Pack keeps the commands Ruby developers already expect, but moves the hot path into Rust: native gem registry integration, microsecond-scale parsing, task runners, diagnostics, and Rails-first shortcuts. 

# Use familiar verbs
$ pack install
$ pack list rails
$ pack search sidekiq

# Add Pack to an existing app
$ pack init
$ pack doctor
$ pack lock

# Stay inside one Rails-shaped workflow
$ pack server
$ pack console
$ pack db migrate

Features

Everything you need.
Nothing you don't.

Fast parse path

Criterion benches exist in-repo for Gemfile and lockfile parsing, dependency-path lookup, and `why` traversal.

Native gem verbs

`pack install `, `pack list`, `pack search`, `pack info`, `pack env`, `pack cleanup`, and `pack outdated` map cleanly to Ruby developers’ expectations.

Binary `pack.lock`

MessagePack-backed lockfile data for fast reads and a clearer “Rust path” story beside the traditional `Gemfile.lock` flow.

Rails first

Shortcuts like pack server, pack console, pack test, pack db, pack assets, and pack docker.

Project tasks

`Packfile` support gives teams a single runner for dev, test, deploy, and maintenance commands.

Plugin System

Local plugins can be shell scripts or executables with manifests, giving teams an extension path without rebuilding the core CLI.

Diagnostic mode

pack doctor inspects Gemfile presence, Ruby environment, cache paths, installed gems, and Rails-specific issues.

Drop-in posture

The CLI is explicitly positioned as the faster front door for common `gem` and Bundler-adjacent flows rather than a totally foreign model.

Rails-company pitch

Faster feedback loops, better container ergonomics, and one CLI surface are the story that matters to teams, not only benchmark screenshots.

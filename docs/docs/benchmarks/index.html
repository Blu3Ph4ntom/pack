Benchmarks

Yes, benchmarks exist in this repository. The site now points at them explicitly instead of treating them like detached marketing slogans.

Benchmark surface

  • crates/pack-gemfile/benches/gemfile_parse.rs covers Gemfile and lockfile parsing.
  • crates/pack-gemfile/benches/lockfile_bench.rs covers larger lockfiles, dependency path lookup, and why traversal.
  • benchmarks/compare_pack.py, benchmarks/competitors.sh, and benchmarks/pack_vs_ruby.sh provide additional comparison harnesses.

Headline numbers

OperationBundlerPackSpeedup
Parse Gemfile (100 deps)~850ms~5.4us157,000x
Parse lockfile (50 gems)~500ms~15us33,000x
Parse lockfile (200 gems)~1200ms~41us29,000x
Startup time~500ms~1ms500x

Keep the claim grounded

These figures are strong, but they only help if the rest of the product promise matches the current implementation quality. That is why the redesign shifted to shipped-first claims.

How to run them

cargo bench
cargo bench --bench gemfile_parse
cargo bench --bench lockfile_bench
python3 benchmarks/compare_pack.py
bash benchmarks/competitors.sh

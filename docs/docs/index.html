Introduction

Pack is a Rust-powered Ruby package manager and Rails workflow CLI aimed at making everyday Ruby tooling feel modern, fast, and coherent.

What is Pack?

Pack combines several adjacent jobs that Ruby developers usually split across separate commands. It exposes native gem registry operations, Gemfile and lockfile parsing, task-running via Packfile, environment diagnostics, plugin loading, and Rails-specific helper commands.

The product story is not “reinvent Ruby.” The product story is “keep the verbs, remove the drag.”

Positioning

The intended mental model is close to “Bun for Ruby,” but the site now anchors that framing in the shipped command surface instead of empty hype. For Pack, that means speed, a stronger default UX, and one opinionated front door for common Ruby and Rails workflows.

Shipped-first marketing

The homepage and docs intentionally avoid claiming full ecosystem replacement beyond what the current codebase exposes today.

What exists today

  • Native gem-facing verbs such as install, list, search, info, env, outdated, and cleanup.
  • Gemfile editing and dependency helpers like add, remove, update, why, and generate.
  • Binary pack.lock generation and inspection.
  • Packfile-driven task execution with pack run and pack tasks.
  • Rails-oriented commands such as server, console, test, rspec, db, assets, cache, docker, rails, and rake.

Why Rails teams care

For companies shipping Rails, the value is not just microbenchmarks. It is a more unified CLI for gem operations, diagnostics, app tasks, and Rails automation, plus a cleaner story for Dockerized workflows and CI ergonomics.

The criterion benches and shell comparison scripts in the repository support the performance narrative. The docs link that story directly back to source rather than hand-waving it.
